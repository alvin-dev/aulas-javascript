O objeto XMLHttpRequest é usado para trocar dados com um servidor.

Send a Request To a Server

Para enviar uma solicitação para um servidor, usamos os métodos open () e send () do objeto XMLHttpRequest:

xhttp.open("GET", "ajax_info.txt", true);
xhttp.send();

GET or POST?

O GET é mais simples e rápido que o POST e pode ser usado na maioria dos casos. No entanto, sempre use solicitações POST quando:

  • Um arquivo em cache não é uma opção (atualize um arquivo ou banco de dados no servidor).
  • Enviando uma grande quantidade de dados para o servidor (o POST não tem limitações de tamanho).
  • Enviando a entrada do usuário (que pode conter caracteres desconhecidos), o POST é mais robusto e seguro que o GET.

The url - A File On a Server

O parâmetro url do método open () é um endereço para um arquivo em um servidor:

xhttp.open("GET", "ajax_test.asp", true);

O arquivo pode ser qualquer tipo de arquivo, como .txt e .xml, ou arquivos de script do servidor, como .asp e .php (que podem executar ações no servidor antes de enviar a resposta de volta).

Asynchronous - True or False?

As solicitações do servidor devem ser enviadas de forma assíncrona. O parâmetro async do método open () deve ser configurado como true:

xhttp.open("GET", "ajax_test.asp", true);

Ao enviar de forma assíncrona, o JavaScript não precisa aguardar a resposta do servidor, mas pode, em vez disso:

  • executar outros scripts enquanto aguarda a resposta do servidor
  • lidar com a resposta depois que a resposta estiver pronta

Synchronous Request

Para executar uma solicitação síncrona, altere o terceiro parâmetro no método open () para false:

xhttp.open("GET", "ajax_info.txt", false);
